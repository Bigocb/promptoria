ANSWER TO YOUR QUESTION
=======================

Q: "Is it designed to use Claude specifically or is it possible to change 
    AI client used? Would that be in a settings section?"

A: ✅ YES - NOT Claude-specific, and YES - there's a Settings section!

The system supports MULTIPLE AI PROVIDERS that are easily configurable
through a dedicated Settings page.


SUPPORTED PROVIDERS
===================

🔴 OpenAI
   ├─ GPT-4 ($0.03-0.06/M tokens)
   ├─ GPT-4 Turbo ($0.01-0.03/M tokens)
   └─ GPT-3.5 Turbo ($0.50-1.50/M tokens)
   Endpoint: https://api.openai.com/v1
   Requires: API Key from platform.openai.com

🤖 Anthropic (Claude) [NOT THE ONLY OPTION!]
   ├─ Claude 3 Opus ($0.015-0.75/M tokens)
   ├─ Claude 3 Sonnet ($0.003-0.15/M tokens)
   └─ Claude 3 Haiku ($0.0025-0.0125/M tokens)
   Endpoint: https://api.anthropic.com/v1
   Requires: API Key from console.anthropic.com

🌊 Cohere
   ├─ Command ($0.001-0.001/M tokens)
   └─ Command Light ($0.0003-0.0006/M tokens)
   Endpoint: https://api.cohere.ai/v1
   Requires: API Key from cohere.com

☁️ Azure OpenAI
   ├─ GPT-4
   └─ GPT-3.5 Turbo
   Endpoint: https://{resource}.openai.azure.com
   Requires: Azure endpoint + API key

🦙 Ollama (Local/Free)
   ├─ Llama 2 (Free)
   ├─ Mistral (Free)
   └─ Neural Chat (Free)
   Endpoint: http://localhost:11434
   Requires: Nothing (runs locally)


SETTINGS PAGE
=============

Location: http://localhost:3000/settings

Features:
  ✅ Add multiple provider configurations
  ✅ Give each a name and description
  ✅ Set default provider for workspace
  ✅ Configure per-provider defaults (temperature, tokens, etc.)
  ✅ Test connections before using
  ✅ Track usage and costs per provider
  ✅ Enable/disable without deleting
  ✅ Override per prompt or use defaults


WORKFLOW
========

1. SETUP (First Time)
   Go to /settings
   Click "Add Provider"
   Select OpenAI or Claude or any provider
   Enter API key
   Select default model
   Click "Test" to verify
   Save

2. ADD MORE PROVIDERS (Optional)
   Can have multiple configs of same provider
   "Production OpenAI", "Testing OpenAI", "Backup Claude"
   Each has own key and settings
   Easy to switch between them

3. USE IN PROMPTS
   Create/edit prompt
   Choose provider:
     • Use workspace default (OpenAI)
     • Use specific config (Claude Sonnet)
     • Manual selection in Test Runner
   Test prompt with selected provider
   See output, tokens, cost

4. SWITCH PROVIDERS
   Just change default in Settings
   All future prompts use new provider
   No code changes needed

5. MONITOR COSTS
   Settings shows per-provider stats:
     • Total cost spent
     • Total tokens used
     • Request count
     • Error count


CONFIGURATION DETAILS
=====================

When adding a provider, configure:

  Name:           "Production OpenAI" or "Claude Testing"
  Description:    "Main account" or "Testing cheaper models"
  Provider:       Dropdown (OpenAI, Claude, etc.)
  API Key:        Encrypted storage, never exposed
  Default Model:  GPT-4, Claude Opus, Cohere Command, etc.
  Temperature:    0-2 (creativity/randomness)
  Max Tokens:     100-4000 (response length)
  Top P:          0-1 (diversity)
  Public:         Can team members use this?


DATABASE
========

New table: AIProviderSettings

  id                    -- Unique ID
  workspace_id          -- Which workspace
  provider              -- openai | anthropic | cohere | azure | ollama
  name                  -- User-friendly name
  api_key_encrypted     -- Never stored plaintext
  default_model         -- Default model to use
  default_temperature   -- Default temp
  default_max_tokens    -- Default max tokens
  default_top_p         -- Default top P
  is_default            -- Use for prompts without override
  is_enabled            -- Quick on/off
  is_public             -- Team access?
  total_tokens_used     -- Running tally
  total_cost            -- Running tally
  request_count         -- How many times used
  error_count           -- Failed requests
  tested_at             -- Last successful test


API ENDPOINTS
=============

POST /api/settings/providers
  Create a new provider configuration
  Body: {provider, name, api_key, default_model, ...}

GET /api/settings/providers
  Get all configured providers
  Query: workspace_id

GET /api/settings/providers/default
  Get the default provider
  Query: workspace_id

PUT /api/settings/providers/:id/default
  Set as default provider

POST /api/settings/providers/:id/test
  Test connection to this provider

PUT /api/settings/providers/:id
  Update configuration

DELETE /api/settings/providers/:id
  Delete configuration

USE CASES
=========

Case 1: Cost Optimization
  • Using Claude Opus (expensive)
  • Add Claude Sonnet (cheaper)
  • Test same prompt with both
  • Compare output quality + cost
  • Switch to Sonnet if comparable

Case 2: Emergency Failover
  • Primary provider (OpenAI) has outage
  • Switch default to Claude in Settings
  • All prompts immediately use Claude
  • No code changes, no downtime

Case 3: Multi-Provider Testing
  • Create 3 configs: GPT-4, Claude, Cohere
  • Test same prompt with each
  • See quality, latency, cost
  • Keep best one as default

Case 4: Team Sharing
  • Admin sets up OpenAI config
  • Marks as is_public: true
  • Whole team can use
  • Individual usage tracked

SECURITY
========

API Keys:
  ✓ Encrypted before storage (AES-256)
  ✓ Never logged or exposed
  ✓ Only decrypted when needed
  ✓ Can rotate without losing config

Access Control:
  ✓ Per-workspace isolation
  ✓ Per-config public/private flag
  ✓ Only enabled configs can be used

Best Practices:
  ✓ Use environment variables
  ✓ Rotate keys periodically
  ✓ Different keys for different environments
  ✓ Never commit keys to Git
