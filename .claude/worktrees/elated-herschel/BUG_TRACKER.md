# Bug Tracker

We use **GitHub Issues** to track bugs, feature requests, and questions for Promptoria.

## How to Report a Bug

1. Go to the [GitHub Issues page](https://github.com/Bigocb/promptoria/issues)
2. Click "New Issue"
3. Use one of the issue templates (Bug Report, Feature Request, etc.)
4. Fill in the required information:
   - **Title**: Clear, concise description
   - **Description**: Steps to reproduce, expected behavior, actual behavior
   - **Environment**: OS, browser, app version
   - **Attachments**: Screenshots or error logs if relevant

## Bug Labels

- `bug` - Confirmed bug
- `critical` - Blocks usage or causes data loss
- `high` - Significant impact on user experience
- `medium` - Minor issues or workarounds exist
- `low` - Polish or cosmetic issues

## Issue Priorities

### Critical 🔴
- Data loss issues
- Complete feature non-functionality
- Security vulnerabilities

### High 🟠
- Major functionality broken
- Significant performance issues
- User-facing errors

### Medium 🟡
- Non-critical bugs
- Partial feature issues
- Minor UI/UX problems

### Low 🟢
- Cosmetic issues
- Nice-to-have improvements
- Documentation gaps

## Contributing

When fixing a bug:
1. Reference the issue number in your commit message (e.g., `Fix #123`)
2. Link your PR to the issue
3. Provide clear explanation of the fix
4. Add tests if applicable

Thank you for helping make Promptoria better! 🚀
