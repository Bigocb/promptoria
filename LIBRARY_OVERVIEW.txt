DISCOVERY & ORGANIZATION
========================

SEARCH
  Full-text search across:
  • Name
  • Description
  • Tags
  • Content (optional)

FILTERS
  • Type: prompt, skill, instruction, template, snippet
  • Category: marketing, technical, content, seo, email, etc.
  • Difficulty: beginner, intermediate, advanced
  • Tags: Any custom tags
  • Public/Archived: Status

SORTING
  • Most Used (highest use_count)
  • Highest Rated (best ratings)
  • Recently Used (last_used timestamp)
  • Newest Created (created_at)


QUALITY METRICS
===============

⭐ RATINGS
  • 1-5 star system
  • Calculated from comments
  • Shows community consensus
  • Helps discover best items

📊 USAGE STATS
  • use_count: How many times used
  • last_used: When was it last accessed
  • success_rate: % of successful executions
  • avg_latency: Average execution time (ms)
  • avg_cost: Average cost per execution ($)

💬 COMMENTS
  • Team feedback
  • Improvement suggestions
  • Use cases and examples
  • Issue reports


API ENDPOINTS
=============

GET /api/library
  Parameters:
    action=list|get|stats|recommended
    type=prompt|skill|instruction|template|snippet
    category=marketing|technical|seo|etc.
    tags=tag1,tag2,tag3
    search=query
    difficulty=beginner|intermediate|advanced
    workspace_id=id
  
POST /api/library
  Actions:
    create    - Create new item
    fork      - Duplicate an item
    comment   - Add comment/rating
    usage     - Record metrics
    import    - Import from JSON
    export    - Export to JSON

PUT /api/library/:id
  Update: name, description, content, tags, etc.

DELETE /api/library/:id
  Archive: soft delete (preserves history)


DATABASE SCHEMA
===============

LibraryItem (Main table)
  id, workspace_id, type
  name, description, category, subcategory
  content, instructions, example_output
  tags[], difficulty
  use_count, last_used, avg_latency, avg_cost, success_rate
  rating, is_public, is_archived, is_template
  version_number, changelog
  parent_id, related_ids[]
  created_by, created_at, updated_at

LibraryComment
  id, library_item_id, author, comment, rating, created_at

LibraryFork
  id, source_id, fork_id, forked_by, created_at

LibraryCollection (Optional)
  id, workspace_id, name, description, item_ids[], created_by


WORKFLOW
========

CREATE → TEST → SAVE → DISCOVER → REUSE → IMPROVE

Step 1: CREATE
  ✓ User creates prompt using snippets
  ✓ Composes template with variables

Step 2: TEST
  ✓ Test in Test Runner
  ✓ See output, tokens, cost, latency
  ✓ Record performance metrics

Step 3: SAVE TO LIBRARY (NEW!)
  ✓ Save tested prompt to library
  ✓ Set name, description, category
  ✓ Add tags, difficulty, instructions
  ✓ Include example output

Step 4: DISCOVER (NEW!)
  ✓ Team members find it
  ✓ Search in Library Browser
  ✓ See rating, usage, performance
  ✓ Read comments and reviews

Step 5: REUSE (NEW!)
  ✓ Fork for customization
  ✓ Or use as snippet in new prompts
  ✓ Performance auto-tracked

Step 6: IMPROVE (NEW!)
  ✓ Rate and comment
  ✓ Suggest improvements
  ✓ Version updated
  ✓ History preserved
